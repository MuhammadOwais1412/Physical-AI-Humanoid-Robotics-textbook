# Glossary

This glossary defines key terms used throughout the Physical AI & Humanoid Robotics textbook.

## A

**Action Space** - The set of all possible actions that a robot can perform in its environment.

**Affordance** - The possibility of an action that an object provides to an agent; for example, a handle affords grasping.

**Artificial General Intelligence (AGI)** - A type of AI that can understand, learn, and apply knowledge across a wide range of tasks at a level comparable to human intelligence.

## B

**Behavior Tree** - A hierarchical structure used to organize and control the behavior of autonomous agents, commonly used in robotics and game AI.

**Bipedal Locomotion** - The act of walking on two legs, a key capability for humanoid robots.

## C

**Computer Vision** - A field of artificial intelligence that trains computers to interpret and understand the visual world.

**Convolutional Neural Network (CNN)** - A class of deep neural networks commonly used for analyzing visual imagery.

**Cognitive Architecture** - An explicit specification of the cognitive components and processes required to perform tasks in a domain.

## D

**Deep Learning** - A subset of machine learning based on artificial neural networks with representation learning.

**Digital Twin** - A virtual representation of a physical object or system that spans its lifecycle, is updated with real-time data, and uses simulation, machine learning and reasoning to help decision making.

**Docker** - A set of platform as a service products that use OS-level virtualization to deliver software in packages called containers.

## E

**Embodied AI** - Artificial intelligence that involves a physical body interacting with the real world, as opposed to purely software-based AI.

**Encoder** - A device that converts mechanical motion into electrical signals for measurement of position, velocity, or acceleration.

## F

**Forward Kinematics** - The use of kinematic equations to compute the position of the end-effector from specified values of joint parameters.

## G

**Gazebo** - A 3D simulation environment for robotics that provides realistic physics, high-quality graphics, and convenient programmatic interfaces.

**Generative Pre-trained Transformer (GPT)** - A type of large language model that uses the transformer architecture for natural language processing tasks.

**Geometric Deep Learning** - A framework for deep learning that deals with non-Euclidean data such as graphs and manifolds.

## H

**Human-Robot Interaction (HRI)** - The study of interactions between humans and robots, focusing on design, development and evaluation of robotic systems for human use.

**Humanoid Robot** - A robot with a physical body that has a kinematic structure similar to that of a human.

## I

**Inverse Kinematics** - The mathematical process of calculating the variable joint parameters needed to place the end of a kinematic chain, such as a robot manipulator or animation character, in a given position and orientation.

**Isaac Sim** - NVIDIA's next-generation robotics simulation application and synthetic data generation tool that provides a highly realistic physics simulation environment.

## L

**Large Language Model (LLM)** - A language model consisting of a neural network with many parameters, trained on vast text corpora to understand and generate human-like text.

**LiDAR** - Light Detection and Ranging; a remote sensing method that uses light in the form of a pulsed laser to measure distances.

**Long Short-Term Memory (LSTM)** - A type of recurrent neural network (RNN) architecture that can learn and remember over long sequences.

## M

**Manipulator** - A robot arm or other device that can be used to manipulate objects in the environment.

**Middleware** - Software that provides common services and capabilities to applications beyond what's offered by the operating system.

**Model Predictive Control (MPC)** - An advanced method of process control that uses a model of the system to predict future behavior and optimize control actions.

## N

**Navigation Mesh (NavMesh)** - A data structure that is used to aid pathfinding in a 3D environment, particularly in robotics and game development.

**Neural Radiance Fields (NeRF)** - A method for synthesizing novel views of complex 3D scenes based on a sparse set of 2D images.

**Non-player Character (NPC)** - In robotics simulation, virtual agents that populate the environment and interact with the robot.

## O

**OpenAI** - An artificial intelligence research laboratory consisting of the non-profit OpenAI Inc. and its for-profit subsidiary OpenAI LP.

**OpenCV** - An open-source computer vision and machine learning software library.

**Omnidirectional Wheels** - Specialized wheels that allow a robot to move in any direction, including sideways, without changing the orientation of the platform.

## P

**Path Planning** - The computational problem of finding a valid path from a start point to a goal point while avoiding obstacles.

**Perception Pipeline** - A sequence of processing steps that transform raw sensor data into meaningful information about the environment.

**Point Cloud** - A set of data points in space, typically representing the external surface of an object, often generated by 3D scanners.

**Prompt Engineering** - The practice of designing and refining inputs to guide language models toward desired outputs.

## R

**Recurrent Neural Network (RNN)** - A class of artificial neural networks where connections between nodes form a directed graph along a temporal sequence, allowing it to exhibit temporal dynamic behavior.

**Robot Operating System (ROS)** - A flexible framework for writing robot software that provides a collection of tools, libraries, and conventions for creating robot applications.

**ROS 2** - The second generation of the Robot Operating System, designed to be a production-ready, flexible, and distributed system.

**Reinforcement Learning** - A type of machine learning where an agent learns to make decisions by performing actions in an environment to maximize cumulative reward.

**Riemannian Motion Policies (RMPs)** - A geometric approach to learning robot control policies that combines task specifications with Riemannian geometry.

## S

**Simultaneous Localization and Mapping (SLAM)** - The computational problem of constructing or updating a map of an unknown environment while simultaneously keeping track of an agent's location within it.

**State Estimation** - The process of estimating the internal state of a dynamic system from measurements that are made over time.

**System-on-Chip (SoC)** - An integrated circuit that integrates all components of a computer or other electronic system into a single chip.

## T

**Teleoperation** - The remote operation of a robot by a human operator.

**Transformer Architecture** - A deep learning model that uses self-attention mechanisms to process sequential data, widely used in natural language processing.

**Trajectory Optimization** - The process of finding a time-parameterized path that optimizes a given cost function while satisfying constraints.

## U

**Unity** - A cross-platform game engine that is widely used for creating 3D simulations and virtual environments for robotics.

**Universal Robot Description Format (URDF)** - An XML format for representing a robot model, including its links, joints, and other properties.

**Unstructured Environment** - An environment that has not been specifically designed or modified to facilitate robot operation, as opposed to a structured environment like an assembly line.

## V

**Vision-Language-Action (VLA)** - A paradigm that combines visual perception, language understanding, and physical action in robotic systems.

**Visual Servoing** - A technique that uses visual feedback to control the motion of a robot.

**Voice Activity Detection (VAD)** - A technology that detects the presence or absence of human speech in an audio signal.

## W

**Whisper** - OpenAI's automatic speech recognition (ASR) system that is trained on a large dataset of diverse audio and is capable of performing multilingual speech recognition and translation.

**Whole-Body Control** - A control approach for humanoid robots that considers all degrees of freedom simultaneously to achieve multiple tasks.

## Y

**Yaw** - The rotation of a robot around its vertical axis, typically measured in radians or degrees.

## Z

**Zero-shot Learning** - The ability of a model to perform tasks it has not been explicitly trained on, using knowledge transferred from related tasks.
